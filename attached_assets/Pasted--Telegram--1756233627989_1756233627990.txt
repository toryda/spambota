🎯 Цель

Создать приложение, которое позволяет подключать аккаунты Telegram, выбирать папки с чатами/группами/каналами, и автоматически рассылать по ним подготовленные сообщения с заданными интервалами.
Софт должен быть устойчив к ошибкам (логи, диагностика), и удобен в использовании.

⚙️ Функции
1. Подключение аккаунтов

Авторизация по api_id и api_hash.

Поддержка нескольких аккаунтов.

У каждого аккаунта — свой session (хранение локально).

Возможность задать прокси (SOCKS5/HTTP) индивидуально для каждого аккаунта.

2. Работа с папками

При подключении аккаунта софт подтягивает список папок (Dialog Filters).

Пользователь выбирает, в какой папке запускать рассылку.

В каждой папке софт автоматически разделяет:

✅ Каналы/группы/чаты, куда можно писать.

❌ Те, куда писать нельзя (нет прав, чат только для чтения, удалённый канал и т.п.).

3. Работа с сообщениями

Пользователь задаёт текст или несколько вариантов сообщений (софт выбирает случайный для антиспама).

Поддержка:

Текста

Эмодзи

Unicode-шрифтов

Медиа (фото/видео/документы) — по возможности

4. Настройки рассылки

Интервал отправки: настраиваемый диапазон (например, 20–60 секунд случайно).

Ограничение сообщений на аккаунт (например, не более 100 в сутки).

Время активности (например, с 09:00 до 22:00).

Возможность включить несколько аккаунтов для параллельной рассылки.

5. Логирование и диагностика

Встроенный лог:

Куда отправлено ✅

Где ошибка ❌ (например, "нельзя писать в канал", "доступ запрещён").

При ошибке:

Софт предлагает 3 варианта:

Отписаться от проблемного чата.

Перейти в канал/группу (ссылка кликабельная).

Игнорировать (пропустить в следующий раз).

В логах отображается статистика: сколько сообщений отправлено каждым аккаунтом.

Возможность выгрузки логов (CSV/JSON).

6. Интерфейс

Минимум консоль, максимум удобства.

Вариант 1: простой GUI (вкладки: Аккаунты / Сообщения / Папки / Логи).

Вариант 2: если GUI сложно, то удобная CLI-версия с меню и выбором папок.

🚀 Техническая реализация

Язык: Python 3.10+

Библиотека: Telethon

Конфигурация в JSON/YAML (для аккаунтов, прокси, таймингов).

Хранение сессий в папке /sessions/.

Логи в SQLite или JSON.

📊 Пример сценария

Пользователь подключает 3 аккаунта.

Софт подтягивает папки у каждого → выбирает папку “Работа”.

Софт сканирует папку и делит чаты: куда можно писать и куда нельзя.

Пользователь загружает сообщение и ставит интервал 30–90 сек.

Рассылка запускается → софт ведёт лог.

Если аккаунт упирается в ошибку (например, чат read-only) → в логах выводится:

❌ Ошибка: нельзя писать в @group123

 Отписаться

 Перейти

 Игнорировать

🛠 Доп. фишки (по желанию)

Ротация аккаунтов (если один словил лимит — переходит на другой).

Поддержка inline-кнопок.

Автообход антиспама: задержки, рандомизация текста.

Веб-панель для управления (Flask/FastAPI).

👉 Такой софт у тебя будет именно «продуманный спамер»: подключаешь акк → выбираешь папку → задаёшь сообщение и тайминги → он сам сортирует, спамит и логирует.
